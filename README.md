# ***```Словарик по git```*** 
****     
## **Чтобы авторизироваться вводим команды:**
 
```
git config --global user.name "ваше имя в Github" 
```
```
git config --global user.email "почта"**
```
******
## **Основные команды:**

* выводит путь к текущему рабочему каталогу
```
 pwd
```
* создать новый репозиторий 
```
git init
```
* показывает все файлы в директории
```
ls -la 
```
* сохранение изменений
```
git add . 
```
* фиксация изменений и записи их в репозитории
```
git commit -m "initial" 
```
* состояние раб.каталога
```
git status
```
* история коммитов
```
git log 
```
* отображение объекта в простом и понятном виде
```
git show
```
- выгрузка содержимого локального репозитория в удаленный
```
git push
```
 - извлекает и загружает содержимое из удаленного репозитория в локальный
 ```
git pull
```
 - клонирует удаленный репозиторий в локльную папку
 ```
git clone
```
- <font color="brown">игнорирование файлов</font> 
```
gitignore 
```
* переключение между ветками
```
git checkout 
```
- создание ветки
```
git checkout -b 
```
- слияние одной или нескольких веток в текущую
```
git merge 
```
- <font color="brown">"менеджер веток": создает новые, удаляет и переименовывает их</font>
```
git branch
``` 
* временное сохранение всех незакоммиченных изменений для очистки раб.директории без необходимости их коммитить незавершенную работу в новую ветку
```
git stash
```
- выводит скрытые временные изменения (разморозка)
```
git stash pop 
```
- управление списком удаленных репозиториев
```
git remote 
```
- используется, чтобы указать, на каких удаленных конечных точках будут работать команды синхронизации
```git remote -v 
```
- удаление файлов из индекса и рабочей копии
```
git rm 
```
-  сравнения источников данных git — коммитов, веток, файлов и т. д.
```
git diff 
```
- отмена изменений в рабочем каталоге
```
git restore 
```
- удаляет файл из промежуточной области, но оставляет его фактические модификации нетронутыми.
```
git restore --staged 
```
- отмена локальных изменений в репозитории Git
```
git reset
``` 
* Параметр soft говорит о том, что изменения останутся и коммит исчезнет. Изменения будут в индексе
```
git reset --soft
``` 
* По умолчанию
```
git reset --mixed 
```
* Отменяет все незавершенные изменения
```
git reset --hard 
``` 
- создает новые коммиты, но максимально переиспользует содержимое предыдущих, меняет содержимое, сообщение, автора последнего коммита 
```
git commit --amend -m 
```
- позволяет построчно изучить содержимое файла и выяснить, когда и кто внес последние изменения каждой строки
```
git blame 
```
- используется для удаления мусора из рабочей директории
```
git git clean -f 
```
- переименование файлов
```
git mv
```

# ***```Словарик по html```*** 
****     
## **Указание типа текущего документа:**
 
```
<!DOCTYPE>
```
******
## **Основные команды:**
* Контейнер, в котором все содержимое веб-страницы:
```
<html>
```
* Добавить комментарий в код документа:
```
 <!-- --> 
```
* Абзац:
```
<p> 
```
* Создание ссылок:
```
<a> 
```
* Содержание сайта (новости, статьи):
```
<article>
```
* Устанавливает жирное начертание шрифта:
```
<b>
```
* Хранение содержимого веб-страницы:
```
<body>
```
* Перевод строки в том месте, где установлен:
```
<br>
```
* Создает на веб-странице кнопку:
```
<button>
```
- Создает область, в которой при помощи JS можно рисовать и выводить изображения:
```
<canvas>
```
 - Создает заголовок к таблице:
 ```
<caption>
```
 - Выравнивает содержимое контейнера по центру:
 ```
<center>
```
- Представляет название произведения:
```
<cite>
```
* Задает ширину и другие характеристики одной или нескольких колонок таблицы:
```
<col>
```
- Создает команду в виде переключател, флажка или обычной кнопки:
```
<command>
```
- Добавляет комментарий в код:
```
<comment>
```
- Выделение текста, который был удален в новой версии:
```
<del>
``` 
* Создает список, содержащий названия директорий:
```
<dir>
```
- Стилизация:
```
<div>
```
```
<span>
```
- Акцентирование текста:
```
<em>
```
- Изменение характеристик шрифта (размер, цвет, гарнитура):
```
<font>
```
- Заголовки 1, 2, 3, 4, 5, 6 уровней соответсвенно:
```
<h1>
```
```
<h2>
```
```
<h3>
```
```
<h4>
```
```
<h5>
```
```
<h6>
```
-  Хранение других элементов:
```
<head>
```
- Идентификация верхней части страницы:
```
<header>
```
- Курсив:
```
<i>
``` 
* Изображение:
```
<img>
``` 
* Связь между меткой и элементом формы:
```
<label>
```
* Основное содержимое документа:
```
<main>
``` 
- Список меню:
```
<menu>
```
- Количество колонок, ширина, расстояние ммежду колонок в тексте:
```
<multicol>
```
- нумерованный список:
```
<ol>
```
- Контейнер для хранения нескольких элементов (изображений):
```
<picture>
```
* Определение стилей элементов:
```
<style>
```
* Таблица:
```
<table>
```
* Хранение одной или нескольких строк таблицы:
```
<tbody>
```
* Заголовок документа:
```
<title>
```
* Разметка текста, который должен отличаться стилистически от обычного:
```
<u>
```
* Маркированный список:
```
<ul>
```
* Выделение переменных из комп.программ:
```
<var>
```
